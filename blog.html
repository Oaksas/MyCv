Pros of using relative size( %, em, rem) over pixel.

%, em and rem are relative measurement units of length used in webpages. As webpages could be run on different screen size platforms or devices, using the static and most precise measurement like pixel( px) would not be so convenient

Using pixels as a measurement unit in webpages, most of the time, limits users from experiencing the most from the website they want to surf. They have to use the same screen-size device as the developer in order to see all the same output as the developer. Moreover partially-visually-impaired users could not increase, let’s say, the font size of the page he/she would like to access. This would cause a bias of information among users. The layout is also one problem: as the user with smaller or larger screen-size device might find the webpage layout changing as screen-size changes. The convenient ways to use pixel are: when we want build a website which could be used in a small team on the same screen-sized device, and when we want to preserve some constant spacing among the webpage Element.

- Em – these measurement is taken as a reference to its container. For example, 1 em of a paragraph in div tag is determined by in-proportion to the font size of its parent div.

- Rem – these measurement is taken as a reference to the root element which is the HTML itself. Usually most browsers use the HTML size of 16px so 1Rem = 16px.

- % - puts the measurement value relative to the parent.

How Does The Browser Interprete Pixels ?

CSS pixel - is the height and width of a dot that can be seen comfortably without strain.1 CSS pixel is equal to 1 physical pixel at 96 DPI ( Pixel Density) located at arm’s length from the users view point.

CSS pixels are basically not actual pixels. They are created for the convenience of the developers. Basically the CSS pixels change according to the zoom level of the user. On the contrary the device pixel, which is the actual pixel, does not change under any circumstances. So CSS pixels are relative to the user’s experience.

So according to my conclusion, specifying the size of an element's size with a pixel larger than browser’s default pixel would make the browser act like the second device pixel in which the newly specified element is situated in proportion to the browser’s full pixel size (mostly it is 16px).

In full screen mode (100% zoom), the browser displays the full element as the developer sees it. For example, if the browser's font-size is 16px and the new element inside the HTML is about 32px, the size of the newly added element will look like twice the size of the fonts of the browser.

In using absolute measurement, like pixel (px), the zooming effect will not affect the size of the Element. As a result the users might experience a problem while adjusting their views.

To overcome these problem, programmers use dynamic or relative measuring units like em, rem, %, vm and many others. Dynamic measuring units give the browser to interpret the document in proportion to the parent or the root specified.

Newly Added CSS Element

Selectors

Element[attribute=”value”]   =   target an element with an attribute with a value “value”.

Element[attribute^ = “value”]   =  target and element that starts with “value”

Element[attribute$^=”value”]   =  target and element that finishes with “value”

Element:nth-child(nth)   =  target the element at nth position of parent.

Element:last-child   =  target the element at the last position of parent.

Element:last-of-type   =  target the last type of Element.

Element:only-of-type   =  target the only child element of type Element.

Element:empty    =   targets the element with no child.

Element:dispabled    =   targets the element with disabled elements.

:not(Element)    =   targets all other HTML elements without Element.

::selection    =   targets the element that the user selects.
